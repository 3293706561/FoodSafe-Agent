# 法规数据核验清单

生成日期：2026-07-12

## 结论

当前知识库可用于作品集 MVP 演示：30 条法规条款、30 条黄金问题均已结构化，且本地规则检索可以为全部黄金问题返回引用依据。

但它还不能作为正式法规库使用。下一轮要优先处理两类问题：一是把二级来源替换为一手官方来源，二是逐条核验 GB 标准的实施日期、过渡期和条款原文位置。

## 已完成的本地核验

- `data/regulations.json`：30 条可检索法规条款，字段完整。
- `data/regulations.full.json`：保留 TeleAgent 收集的完整结构化字段。
- `data/golden_questions.json`：30 条黄金测试问题，字段完整。
- `tests/test_golden_questions.py`：已接入自动化测试，要求 30 条问题全部有引用，且风险等级与预期一致。
- 网页端 `web/regulations-data.js` 已同步同一份法规快照。

## 高优先级人工复核项

1. 食品安全法条款版本：当前库写有“2015-10-01（2018年修正）”，需核验最新修正版本及第 67、71、125 条是否表述一致。
2. GB 7718-2025 实施安排：当前库统一写 `2027-03-16`，需核验过渡期、旧版标签使用规则和库存产品处理规则。
3. GB 28050-2025 实施安排：同样需核验 `2027-03-16` 与过渡期。
4. GB 2760-2024：当前来源不是一手标准文本，应替换为官方标准检索平台或发布公告。
5. 《食品标识监督管理办法》：当前来源是市监总局附件，需要确认发布日期、实施日期、附件 URL 稳定性。
6. A012 “标准年代号可以不标示”：已根据官方问答补入演示数据，但需要在正式库里记录精确问答位置。

## 可继续扩充的方向

- 给每条法规增加 `verification_status`：`verified`、`needs_manual_check`、`secondary_source_only`。
- 给每条法规增加 `source_quote_location`：网页标题、问答序号、条款号或附件页码。
- 对黄金问题增加 `expected_clause_ids`，让测试不只检查风险等级，还检查是否引用了正确条款。
