# 简历描述草稿

## 一句话版

食品安全法规智能体（FoodSafe Agent）：面向预包装食品标签合规的可溯源问答 MVP，支持法规检索、风险分级、整改建议、引用展开和知识库外拒答。

## 项目经历版

**食品安全法规智能体 FoodSafe Agent｜个人项目｜Python / JavaScript / RAG / 规则引擎**

- 面向食品企业标签审版与法规咨询场景，构建“结构化法规库 + 轻量检索 + 规则风险判断”的离线 MVP，覆盖预包装食品强制标示事项、配料表、营养标签、添加剂、执行标准等典型问题。
- 整理 30 条结构化法规条款与 30 条黄金测试问题，为每条回答输出风险等级、整改建议、法规/标准名称、条款摘要和来源链接；对知识库外问题设计拒答机制，避免无依据合规结论。
- 开发静态 Web Demo 和 CLI 工具，支持标签问题输入、风险分级、引用条款展开和按风险筛选；无需外部 API，可部署到 GitHub Pages，适合面试现场稳定演示。
- 建立自动化测试，覆盖核心样例、黄金问题字段完整性、风险等级与引用返回；沉淀法规数据核验清单，区分官方来源、二级来源和待人工复核项。

## 更偏 AI 应用岗位的版本

- 从食品标签合规业务出发，自建可溯源法规问答 Agent MVP，将“用户问题 -> 条款检索 -> 风险判断 -> 整改建议 -> 引用依据”串成可复现闭环；相比通用 RAG，更强调拒答、引用一致性和人工复核边界。
- 设计结构化法规数据 schema 和黄金测试集，使用确定性规则承载合规判断，预留 LLM 只做基于检索证据的表达层，降低模型幻觉对合规场景的影响。

## 更偏国企数字化 / FDE 的版本

- 面向食品企业质量合规流程，构建标签法规初筛工具，将法规条款、审版问题、风险分级和整改动作结构化，支持质量人员快速定位依据并形成可审计记录。
- 采用无外部依赖的 Python + 静态 Web 架构，降低部署复杂度；后续可扩展为企业内网 FastAPI 服务，接入法规更新、标签 OCR、工单流转和人工复核机制。

## 面试时可量化指标

- 法规条款：30 条结构化条款。
- 黄金问题：30 条。
- 自动化测试：5 个测试项，覆盖核心样例和黄金问题。
- 演示方式：CLI + 静态 Web Demo。

不要虚构准确率。当前更准确的说法是：“在当前 30 条黄金问题上，本地规则版可以全部返回引用，并通过风险等级回归测试。”
